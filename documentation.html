Usage example
Here is a full example how to be used:
Include the required javascripts and CSSs.
<html>
<head>
<script type='text/javascript' src="js/jquery-1.9.1.0.min.js"></script>
<script type='text/javascript' src="js/bootstrap.js"></script>
<script type='text/javascript' src="js/bootstrap-slider.js"></script>
<script type='text/javascript' src="js/bootstrap-multiselect.js"></script>
<script type='text/javascript' src="js/bootstrap-dataprospector.js"></script>
<script type='text/javascript' src="js/cpu_items.js"></script>
<link href="css/bootstrap.css" rel="stylesheet">
<link href="css/bootstrap-slider.css" rel="stylesheet">
<link href="css/bootstrap-multiselect.css" rel="stylesheet">
<link href="css/open-iconic-bootstrap.css" rel="stylesheet">
<link href="css/bootstrap-dataprospector.css" rel="stylesheet">
</head>
<body>
<div class="some_container_div">
<div id="here_comes_the_table" class="some_custom_styling">
</div>
</div>
</body>
<script type="application/javascript">
var cpu_dt = $("#here_comes_the_table").dataprospector({data:cpu_items});
</script>
</html>
 The code marked as
dependency
can be downloaded from the their own website or github repository.

Options
List of all the available option variables and explained
Here is a list, how to use the library.
Available options for array
is applied when the data as an array of Array:
[
[ "www.pcx.hu", "AMD FX-4300 FD4300WMHKBOX processzor",...],
[...],...
]
Available options for structure
is applied when the data as an array of Structure:
[
{ "domain":"ipon.hu", "name":"AMD A4-4020 3.20GHz FM2 BOX",...},
{...},
...
]
Option name Available options for data: Array Available options for data: Structure Description
data [ [ "www.pcx.hu", "AMD FX-4300 FD4300WMHKBOX processzor", ...], [...], ...] [{ "domain":"ipon.hu", "name":"AMD A4-4020 3.20GHz FM2 BOX", ...},{...},...] The data is rendered from here. If not set, or the import_first_line_headers = true, then it will try to process the pointing table for data
clone_data true/false true/false Default is false. Should the plugin clone the data? If not cloned, it will make changes to the original dataset: new columns (advanced filters, selected, filtered_out columns)
headers ['domain','name','price'] [0,1,2] List of headers, mostly for internal operations.
header_names {domain:"Shop", name:"Product name", price:'Price", ...} ["Shop", "Product name", 'Price", ...] OR {0:"Shop", 1:"Product name", 2:'Price", ...} Overwrites the names of the headers (in headers and filters)
header_types {domain:'enum', name:'string', price:'range', mem_speed:'array_range',...} ['enum','string','range','array_range',...] Override the filter type, the plugin automatically estimates them from the data.
filters ['domain','name','price'] [0,1,2] List of columns, to enable the filtering on. If empty, it will use all of the columns. Set to false, to turn off false. Columns, that are not listed in the headers or filters will not be visible
external_filters [
{name: "price", type: "range", min:10,max:200},
{name: "name", type: "string", string:"findme"},
{name: "family", type: "options", string:"athlon"},
...
] 		[
{name: "price", type: "range", min:10,max:200},
{name: "name", type: "string", string:"findme"},
{name: "family", type: "options", string:"athlon"},
...
] 		Filters that are applied from javascript, or from menucontext. External_filter array will be overwritten, so if updated, the old external filters should be resent with the new ones. Menu context's filter like this will add a new filter to the list. Calling the externalFilters(array_of_filters) will replace all the external filters. (This does not affects the filters, and the advanced filters modified by the user)
advanced_filters [
{name: "price / power", formula: "price/power", unit: "Ft / W"},
{name: "price/score", formula: "price/bench_score", unit: "Ft/Sc"},
...
] 		[
{name: "price / power", formula: "2/3", unit: "Ft / W"},
{name: "price/score", formula: "2/3", unit: "Ft/Sc"},
...
] 		If it's true (anything else, than false), than the advanced filters will be available, and the filter, and the content the filter depending will be rendered. This means, a new column will be created in the data, with the formula given.
header_shown ['domain','name','price'] [0,1,2] What headers should be displayed. If the filters are not changed, these will be shown as filters as default.
filters_opened true/false true/false The filters side panel should be opened after initialization.
filter_animation_time number in ms number in ms How long should the filter open / close animations do. Default is 1000 ms.
untis {price:"Ft",power:"W",freq:"Mhz",...} [3:"Ft",5:"W",6:"Mhz",...} The units it should display in: listing, details and filter limit values. Will append the unit at the end of the value
items_per_page 25 25 How many item should be printed in a single page
sort_by 'name' 1 The default sort column. Can be changed later. When changing later manually, the dataprospector.sortBy() should be called after.
sort_reverse true/false true/false The data sorting direction (ascending / descending). When changing later manually, the dataprospector.sortBy() should be called after.
item_details_attribute_per_row 2 2 How many attribute should be rendered in the item details element per row.
selection true/false true/false Creates an extra column with checkbox, that can be queried.
import_first_line_headers true/false true/false If the target is an HTML table with existing data to process, and you want to explicitly tell, that the first line is the headers. This is advised, when the header line is the same structure as the rest of the table. tr -> td
context_menu true/false true/false On right-clicking on an item should bring a menu up. Default is false. If true, it will contain:
Filter like these
Reset manual filters
Reset all filters
Select all remaining element
context_menu_download true/false true/false Only valid, if the context_menu is enabled and the selection is also enabled On enabling, it will give several downloading options, which will only download the selected items. Default is false. If true, it will contain:
Download selected as JSON
Download selected as CSV
Download selected as TXT
Download selected as XML (atributes)
Download selected as XML (tags)
Download selected as SQL
no_details true/false true/false Disables the item details when hover over a row for details. Default is false: item details is enabled.
plot_id 'name' / false 1 / false Determines that you want graph / plotting allowed, and set which column is the identifier. That column will be displayed on the graph, when the user hovers over an element.
There are html templates that can be overwritten, but if you want to do that, you should look at the source for more information.
