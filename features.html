Bandwidth and ping friendly:
Data is one time delivered, and the processing done on client side
You send the data once, and the filtering happens in the browser.
There are some cases, where it has more cons than pros, but it can be useful.
When to use
  • You want to save on bandwidth
  • You want to save on server cpu
  • The users take long time with the data
When not to use
  • You have valuable data, and you are afraid, the users will steal it all
  • You have a LOT OF data
  • You want to acquire what the users done with your data

Context menu
You can save the selected items
Context menu only can be enabled, when the depending selected option is also enabled. It will give the users the feature to right click on the data, and get a context menu. It will allow, to clear the filters, setup new external filter based on where the user clicked. This will create a new external filter, based on the column, and the exact value clicked. User can unselect all records, or select the filtered records. There are several different formats for "downloading".

Loading local file
You can use it to process local files easily
You can load a local json, or csv file to find the valuable data you need.
You create an input type="file" element, and set up a listener for a file selected.
It will trigger the creation. It will remove the input element, and replace it with a div and the datacontents.
The json should have to have one of the following format:
array : array = [ ['e','r','t'],['s','d','f'],...]
array : object = [ {name:'e',"tricky-name":'r',description:'t'},{name:'s',"tricky-name":'d',description:'f'}...]
The csv (Coma separated values) has fulfill the following rules:
  • The separator should be , (coma)
  • The first line is set up as headers
  • tricky strings can be quoted ("like this")

Advanced filtering
Easy to setup advanced new columns on depending on existing ones
You can setup custom filtering based on the existing numerical columns.
The result will be new filtering, always slider type.

Try it yourself

Initialise from javascript ...

var mb_dt2 =

var mb_dt = new Dataprospector($("#mbPanel2"), {
data: mb_items,
header_shown: ['domain', 'name', 'price'],
advanced_filters: [
{name: "price / power", formula: "price/power", unit: "Ft / W"},
{name: "price/score",formula: "price/bench_score",unit: "Ft/Sc"},
{name: "price/freq*cores", formula: "price/(freq*cores)", unit: "Ft/MhzCore"}]
});
</script>
... Ether form is accessible, as jquery.databale, as new Dataprospector.. The data will be cloned, so don't worry.
It uses the following input:
advanced_filters is an array ([]), not an object ({})
name
This is what will be displayed in the advanced filter column, and when sorted by, no strict restrictions.

formula
This is what gives the values. It has to be javascript compatible. If you are using other values, they have to declared, before the dataprospector init. These variables will be used only once. For performance dataprospector creates new columns for these values. (It's much cheaper to allocate memory in the browser, than to do a lot of processing on all the filtering :() If you are not using structured data input (Your data does not have names, it's an array), then you have to use column indexes.
Using structured data (json)

...

col_2_ is the 3. column (you know, the sane programming languages are starting to count from 0)

External filtering
External (manual) filtering can be setup
You can filter on the fly with filtering options coming from other sources (other dataprospector)
dataprospector will remember the set filter options, as long, as you are not overwriting them.
Don't forget to set all the filtering options.
If you want to turn it off, just set it to false

Try it yourself

var example_table4;
function example4() {
example_table4 = new Dataprospector($('#example_dataprospector4'), {
data: cpu_items,
header_shown: ['name', 'price', 'cores', 'freq'],
items_per_page: 10
});
example_table4.options.filter_animation_time = 10;
example_table4.toggleShowFilters(example_table4);
setTimeout(function() {
example_table4.toggleShowFilters(example_table4);
example_table4.options.filter_animation_time = 1000;
}, 10);
$("#example_dataprospector4_creator").prop("disabled", true);
return false;
}
 
//  Filter by price < 50 000
function example4a() {
example_table4.externalFilter(
[{name: 'price', type: 'range', min: 0, max: 50000}]
);
}
 
//  Filter by freq > 3 500
function example4b() {
example_table4.externalFilter(
[{name: 'freq', type: 'range', min: 3500, max: Infinity}]
);
}
 
// Filter by price < 50 000 AND freq > 3 500
function example4c() {
example_table4.externalFilter(
[{name: 'price', type: 'range', min: 0, max: 50000},
{name: 'freq', type: 'range', min: 3500, max: Infinity}]
);
}
 
// Reset the external filters
function example4d() {
example_table4.externalFilter(false);
}
How to use it:
Advanced Filter can setup at init, as parameter, or used the .externalFilter function to set it.
name
Name of the column you want to filter. If it's a structure-less (array) data source, you have to use the index number starting from 0
type
Can be one of the following. The other used parameters are depending on the type.
  • range - Will filter a range (min,max), will use the slider's filter function
    • min (number,included)
    • max (number,included)
  • string - Will filter with a string, will use the string filter's function
    • string (string, word-by-word match)
  • array_range - Value is a list of integers ("3600, 4000, 2666, 2133, 2400") (Will filter on the numbers, if any is in the range, it will not filter it out)
    • min (number,included)
    • max (number,included)
  • array_string - Will filter for any match - same as (options)
    • enums (Array of strings)
  • enum - Will filter for any match - same as array_string
    • enums (Array of strings)
